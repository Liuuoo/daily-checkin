# 经典算法深度解析：最长递增子序列 (LIS) 的 $O(n \log n)$ 解法

## 1. 题目描述

给定一个整数数组 `nums`，找到其中最长的**严格递增**子序列（Subsequence）的长度。子序列意味着可以从原数组中删除若干元素（也可以不删除），剩下的元素保持原有相对顺序。

**示例：**
输入：`nums = [10, 9, 2, 5, 3, 7, 101, 18]`
输出：`4`
解释：最长递增子序列是 `[2, 3, 7, 101]`，因此长度为 4。（注意：`[2, 3, 7, 18]` 也是合法的）。

---

## 2. 思路分析

解决该问题通常有两种主流思路，分别对应不同的面试考察点。作为资深工程师，我们不仅要掌握基础的动态规划，更要理解如何通过数据结构优化时间复杂度。

### 方法一：基础动态规划 $O(n^2)$
定义 `dp[i]` 表示以 `nums[i]` 结尾的最长递增子序列长度。
状态转移方程：
$$ dp[i] = \max(dp[j]) + 1 \quad (\text{其中 } 0 \le j < i, \text{且 } nums[j] < nums[i]) $$
若不存在满足条件的 $j$，则 $dp[i] = 1$。
最后返回 `dp` 数组中的最大值。
**缺点**：当 $n$ 达到 $10^5$ 级别时，$O(n^2)$ 会超时（Time Limit Exceeded）。

### 方法二：贪心策略 + 二分查找 $O(n \log n)$ （推荐）
我们需要维护一个辅助数组 `tails`。
**核心定义**：`tails[i]` 存储的是**所有长度为 `i+1` 的递增子序列中，末尾元素的最小值**。

**为什么这样做可行？**
因为对于相同长度的递增子序列，末尾元素越小，后续扩展出更长序列的可能性就越大（贪心性质）。

**算法流程**：
1. 初始化 `tails` 为空列表。
2. 遍历数组中的每个数字 `num`：
   - 如果 `num` 大于 `tails` 中的所有元素，说明我们可以将当前序列延长，直接将 `num` 追加到 `tails` 末尾。
   - 如果 `num` 不能延长当前最长序列，我们尝试用 `num` 替换 `tails` 中第一个大于或等于 `num` 的元素。这不会增加序列长度，但能让该长度的序列末尾变得更小，从而为未来留出更多空间。
3. 使用 `bisect_left` 进行二分查找来定位替换位置。
4. `tails` 的最终长度即为答案。

## 3. Python 代码实现

```python
import bisect
from typing import List

def lengthOfLIS(nums: List[int]) -> int:
    """
    计算最长递增子序列的长度
    时间复杂度：O(n log n)
    空间复杂度：O(n)
    """
    if not nums:
        return 0

    # tails[i] 的含义：长度为 i+1 的所有递增子序列中，
    # 最小的那个末尾元素的值。
    # 这个数组本身一定是单调递增的，因为如果存在长度为 j 的子序列，
    # 那么一定存在长度为 j-1 的前缀子序列，其末尾值必然更小。
    tails = []

    for num in nums:
        # 二分查找：在 tails 中找到第一个大于或等于 num 的位置 idx
        # 这样做的目的是找到可以用 num 来“优化”某个长度的序列末尾
        idx = bisect.bisect_left(tails, num)

        if idx < len(tails):
            # 如果找到了，说明 num 可以成为某个长度子序列的新末尾，
            # 且比原来的末尾更小，更新该位置的值
            tails[idx] = num
        else:
            # 如果 idx == len(tails)，说明 num 比所有已知子序列的末尾都大，
            # 可以直接接在最长子序列后面，长度 +1
            tails.append(num)

    # tails 的长度即为能够构建出的最长递增子序列的长度
    return len(tails)


# --- 测试用例 ---
if __name__ == "__main__":
    test_cases = [
        ([10, 9, 2, 5, 3, 7, 101, 18], 4),
        ([0, 1, 0, 3, 2, 3], 4),
        ([7, 7, 7, 7, 7], 1),
        ([1], 1),
        ([], 0)
    ]

    print("运行结果验证：")
    for i, (input_data, expected) in enumerate(test_cases, 1):
        result = lengthOfLIS(input_data)
        status = "✓" if result == expected else "✗"
        print(f"{status} 案例 {i}: 输入={input_data}, 期望={expected}, 实际={result}")
```

## 4. 时间和空间复杂度分析

### 时间复杂度：$O(n \log n)$
- **外层循环**：遍历输入数组 `nums`，共执行 $n$ 次。
- **内层操作**：每次循环调用 `bisect_left` 对 `tails` 数组进行二分查找。由于 `tails` 的最大长度为 $n$，查找耗时为 $O(\log n)$。
- **总耗时**：$n \times O(\log n) = O(n \log n)$。
- **对比**：相比标准 DP 解法的 $O(n^2)$，在数据量较大时性能提升显著。

### 空间复杂度：$O(n)$
- **额外空间**：`tails` 数组最多存储 $n$ 个元素。
- **系统栈**：无递归调用，无需额外栈空间。

### 代码关键点说明
1. **`bisect_left` 的使用**：这是 Python 标准库提供的二分查找工具，用于找到插入位置以保持有序性。它等价于 C++ STL 中的 `lower_bound`。
2. **不可变性误区**：`tails` 数组并不直接存储最终的最长子序列，它只记录了各长度下的最优“尾部”。例如，`tails` 最终可能是 `[2, 3, 7, 18]`，虽然这可能不是原数组中实际存在的子序列组合，但其**长度**是正确的。如果需要还原具体序列，需要额外记录父节点指针，但这会增加空间复杂度至 $O(n)$ 并略微增加逻辑复杂度。
3. **严格递增处理**：`bisect_left` 遇到相等元素会返回左侧索引，这意味着相等的数会被替换掉，从而保证子序列是严格递增的。如果是非递减（允许相等），应使用 `bisect_right`。
