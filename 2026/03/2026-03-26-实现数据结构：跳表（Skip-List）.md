```python
# 实现数据结构：跳表（Skip List）
# 原理说明：
# 跳表是一种可以高效实现有序集合的数据结构，通过在链表中增加多级索引来实现快速查找、插入和删除。
# 每个节点包含多个“指针”，指向不同层级的下一个节点。查找时，可以从高层级跳过部分节点，加快查找速度。
# 插入时，按照一定概率（通常为0.5）决定节点是否上升到更高层级，形成类似指数级的结构。
# 跳表的时间复杂度为平均 O(log n)（查找、插入、删除），空间复杂度为 O(n log n)（在最坏情况下，但平均是 O(n)）。
# 跳表通常用于实现有序集合，如 Redis 的有序集合和 Java 的 ConcurrencyHashMap 中的跳表结构。

import random

class SkipListNode:
    def __init__(self, level, key, value):
        self.key = key
        self.value = value
        self.forward = [None] * (level + 1)  # 存储各层级的指针

class SkipList:
    def __init__(self, max_level=16, p=0.5):
        self.max_level = max_level  # 最大层级数
        self.p = p  # 升级概率
        self.header = SkipListNode(max_level, -1, None)  # 头节点
        self.level = 0  # 当前跳表的高度

    # 随机生成节点的层级数
    def _random_level(self):
        level = 0
        while random.random() < self.p and level < self.max_level:
            level += 1
        return level

    # 插入操作
    def insert(self, key, value):
        update = [None] * (self.max_level + 1)  # 保存各层级的前驱节点
        node = self.header

        # 从最高层开始向下查找插入位置
        for i in range(self.level, -1, -1):
            while node.forward[i] and node.forward[i].key < key:
                node = node.forward[i]
            update[i] = node

        # 如果该键已存在，更新值（可选）
        if node.forward[0] and node.forward[0].key == key:
            node.forward[0].value = value
            return

        # 生成新节点的层级
        new_level = self._random_level()
        if new_level > self.level:
            for i in range(self.level + 1, new_level + 1):
                update[i] = self.header
            self.level = new_level

        # 创建新节点
        new_node = SkipListNode(new_level, key, value)

        # 将新节点插入到各层级中
        for i in range(new_level + 1):
            new_node.forward[i] = update[i].forward[i]
            update[i].forward[i] = new_node

    # 查找操作
    def find(self, key):
        node = self.header
        for i in range(self.level, -1, -1):
            while node.forward[i] and node.forward[i].key < key:
                node = node.forward[i]
        if node.forward[0] and node.forward[0].key == key:
            return node.forward[0].value
        return None

    # 删除操作
    def delete(self, key):
        update = [None] * (self.max_level + 1)
        node = self.header

        # 查找该键的前驱节点
        for i in range(self.level, -1, -1):
            while node.forward[i] and node.forward[i].key < key:
                node = node.forward[i]
            update[i] = node

        # 如果该键存在
        if node.forward[0] and node.forward[0].key == key:
            deleted_node = node.forward[0]
            for i in range(self.level + 1):
                if update[i].forward[i] == deleted_node:
                    update[i].forward[i] = deleted_node.forward[i]
            # 如果顶层没有节点，减少层级
            while self.level > 0 and self.header.forward[self.level] is None:
                self.level -= 1

    # 打印跳表内容（仅用于调试目的）
    def print_list(self):
        for i in range(self.level + 1):
            node = self.header.forward[i]
            print(f"Level {i}:", end=" ")
            while node:
                print(f"({node.key}: {node.value})", end=" -> ")
                node = node.forward[i]
            print("None")

# 使用示例
if __name__ == "__main__":
    sl = SkipList()

    # 插入
    sl.insert(3, "value3")
    sl.insert(1, "value1")
    sl.insert(5, "value5")
    sl.insert(2, "value2")
    sl.insert(4, "value4")

    # 打印跳表结构
    sl.print_list()

    # 查找
    print("Find 3:", sl.find(3))  # 应输出 "value3"
    print("Find 6:", sl.find(6))  # 应输出 None

    # 删除
    sl.delete(3)
    print("After deleting 3:")
    sl.print_list()
    print("Find 3:", sl.find(3))  # 应输出 None

    # 再插入
    sl.insert(3, "new_value3")
    print("After reinserting 3:")
    sl.print_list()
    print("Find 3:", sl.find(3))  # 应输出 "new_value3"
```

### 复杂度分析

- **插入（Insert）**：平均时间复杂度为 O(log n)。最坏情况下为 O(n)，但通过概率分布，可以保证跳跃层级的平均高度为 log n。
- **查找（Find）**：平均时间复杂度为 O(log n)，因为每次跳跃都可以跳过一半的节点。
- **删除（Delete）**：平均时间复杂度为 O(log n)，需要找到目标节点并更新它所在所有层级的前驱节点的指针。
- **空间复杂度**：平均为 O(n)（每个节点期望有 1/p 个层级，p 为 0.5 时为 2 层级），最坏情况为 O(n log n)。

### 适用场景

跳表非常适合用于实现需要支持快速插入、删除和查找的有序集合，例如：
- 实现 `SortedList` 或 `OrderedSet`。
- 用作 Redis 的有序集合（ZSET）底层实现。
- 在多线程环境中实现非阻塞的有序结构。

跳表相比红黑树而言实现起来更简单，且在并发环境中具有更良好的性能表现。
